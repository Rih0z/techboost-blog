## はじめに

Geekly（ギークリー）はIT・Web・ゲーム業界特化の転職エージェントだ。

「ゲーム業界への転職といえばGeekly」という評判がある一方、Web系エンジニアにも強い。本記事では、Geeklyのサービス内容・強み・デメリットを詳しく解説し、どのエンジニアに向いているかを明確にする。

## Geekly の強み

### 1. ゲーム業界転職に圧倒的な強み

Geekly最大の特徴は**ゲーム・エンタメ業界への転職支援力**だ。

大手ゲーム会社（コナミ・カプコン・セガ等）からモバイルゲーム会社（ガンホー・DeNA等）まで、ゲーム業界の求人を豊富に保有している。

```
Geeklyが特に強いゲーム系企業（例）:
- 任天堂・ソニーインタラクティブエンタテインメント等の大手
- スクウェア・エニックス・バンダイナムコ等のRPGメーカー
- mixi・gloops等のモバイルゲーム会社
- Cygames・Cygames系列スタジオ
```

### 2. Web系・スタートアップ求人も充実

ゲーム業界だけでなく、**Web系企業・スタートアップ・メガベンチャー**の求人も豊富。

- React/Vue/TypeScript等のモダンWeb開発
- バックエンド（Go/Python/Ruby等）
- データエンジニア・ML エンジニア
- テックリード・エンジニアリングマネージャー

### 3. 担当者のIT専門知識

GeeklyのキャリアアドバイザーはIT業界出身者が多く、**技術的な会話ができる担当者**が多い。

```
担当者が対応できること:
- 「TypeScriptのジェネリクスが得意です」という強みの言語化
- ゲームエンジン（Unity/Unreal Engine）経験者への適切なアドバイス
- スタートアップ vs 大手ゲーム会社のカルチャー違いの説明
```

### 4. 非公開求人の多さ

Geeklyで紹介される求人の一部は**一般公開されていない非公開求人**。求人サイトに載っていない好条件の求人にアクセスできる点は大きなメリットだ。

### 5. 内定後のサポートが手厚い

内定後の給与交渉・退職交渉のサポートも充実している。

```
内定後サポート例:
- 「年収600万を提示されたが700万希望」→ 担当者が企業に交渉
- 現職の退職タイミングに合わせた入社調整
- 複数内定時の比較・選択のアドバイス
```

## Geekly のデメリット

### 1. 関東圏外は求人が限られる

Geeklyは**東京中心のサービス**。大阪・福岡にも対応しているが、地方在住・完全リモート希望者は求人選択肢が少なくなる場合がある。

### 2. 未経験・経験浅い場合は対応が限られる

IT業界未経験者の転職には対応していない。**最低でもITの現職経験が1年以上**ないと求人紹介が難しいケースが多い。

### 3. 登録後の連絡頻度が高い場合がある

一部の口コミで「登録後に連絡が多かった」という声がある。転職時期が先でも登録はできるが、担当者からの連絡頻度が合わない場合は率直に伝えることが重要。

## Geekly の利用手順

### Step 1: 登録

1. 公式サイトで氏名・メール・現職情報を入力
2. 登録後2〜3営業日でキャリアアドバイザーから連絡

### Step 2: カウンセリング（オンラインまたは電話）

```
ヒアリング内容:
- 現在の技術スタック・役割
- 転職理由・転職時期
- 志望業界（ゲーム/Web/その他）
- 希望年収・勤務形態
```

### Step 3: 求人紹介・応募

カウンセリング後、担当者から求人が届く。

```
応募の流れ:
1. 担当者からの求人リスト確認
2. 興味のある求人を担当者に伝える
3. 担当者が企業に書類送付
4. 書類通過後、面接日程を調整
```

### Step 4: 面接 → 内定 → 入社

面接対策セッションも依頼可能。

## レバテックキャリア vs Geekly どちらを使うべきか

| 状況 | 推奨 |
|------|------|
| ゲーム・エンタメ業界志望 | **Geekly** |
| Web系・SaaS企業志望 | **レバテックキャリア** |
| 年収600万円以上を狙う | **レバテックキャリア**（求人数が多い） |
| 東京近郊 かつ IT経験あり | **両方併用** が最も効果的 |
| 経験浅い（1〜2年） | **Geekly**（若手向けサポートあり） |

**結論**: どちらか一方でなく、**両方に登録して良い求人・良い担当者を見つけること**が転職成功の近道。

## Geeklyのよくある質問（FAQ）

### Q: ゲーム業界未経験でも登録できますか？

**A:** 登録は可能。ただしゲーム企業への転職には:
- IT業界での開発経験（最低2年以上）
- ゲームへの興味・熱意（ポートフォリオや自作ゲームがあれば強い）

ゲーム専用エンジン（Unity/Unreal Engine）の経験があれば強みになるが、未経験でもWebエンジニアとしてゲーム企業のWebサービス部門に転職できるケースもある。

### Q: 地方在住ですが使えますか？

**A:** 首都圏・大都市圏の求人がメインだが:
- フルリモート求人は全国対応
- ゲーム会社の一部はリモート勤務可
- 転居を前提とする場合は選択肢が広がる

地方在住でリモート限定の場合、選択肢は限られるが可能性はある。

### Q: 登録後にしつこく連絡が来ますか？

**A:** 担当者によって差がある。「今すぐ転職を考えていない」「自分のペースで活動したい」という場合は、最初のカウンセリングで率直に伝えること。連絡頻度の希望を伝えれば対応してもらえる。

## ゲーム業界転職に必要なスキルセット

### Unityエンジニアに求められるスキル

```csharp
// Unity開発で重要なスキルの例:
// 1. C#の基礎から応用（LINQ・async/await）
// 2. Unityの主要コンポーネント理解

using UnityEngine;
using System.Collections;

public class PlayerController : MonoBehaviour
{
    [SerializeField] private float moveSpeed = 5f;
    [SerializeField] private Rigidbody2D rb;

    private Vector2 movement;

    void Update()
    {
        // 入力取得
        movement.x = Input.GetAxisRaw("Horizontal");
        movement.y = Input.GetAxisRaw("Vertical");
    }

    void FixedUpdate()
    {
        // 物理演算で移動
        rb.MovePosition(rb.position + movement * moveSpeed * Time.fixedDeltaTime);
    }
}

// 重要な概念:
// - MonoBehaviour ライフサイクル
// - Object Pooling（ゲーム最適化の必須知識）
// - Scriptable Objects
// - Addressable Assets
// - UniRx / UniTask（非同期処理）
```

### Webエンジニアがゲーム企業のWeb部門に転職するケース

```typescript
// ゲーム企業のWebチームに求められる技術（例）:
// - ゲームポータルサイト・公式サイトの開発

// - 課金・決済システム
```
