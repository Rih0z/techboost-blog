## はじめに

「エンジニアに転職したい」と考える人は年々増加している。IT人材の需要が高まる一方、未経験からのエンジニア転職は決して簡単ではない。

この記事では、**未経験者と経験者それぞれの現実的なロードマップ**を示し、転職成功のために何をすべきかを具体的に解説する。

---

## 2026年のエンジニア転職市場

### 需要の現状

2026年現在、IT人材の需要はかつてないほど高まっている。経済産業省の試算では、2030年には最大79万人のIT人材が不足するとされており、エンジニアの採用市場は売り手市場が続いている。

```
エンジニア求人倍率の推移（参考値）:
2022年: 8.5倍
2023年: 10.2倍
2024年: 11.8倍
2025年: 13.1倍
2026年: 15.0倍（推計）
```

特にAI・クラウド・セキュリティ領域でのエンジニア不足は深刻で、これらのスキルを持つ人材は高い年収が期待できる。

### 未経験者の転職成功率

一方、未経験からのエンジニア転職には現実的な課題がある。

- **成功率**: スクール卒業者の転職成功率は約60〜80%（スクールによる）
- **初年度年収**: 300万〜400万円が一般的（経験者より低め）
- **就職先**: SES企業・受託開発企業が多く、自社開発企業は競争率が高い

「未経験でもエンジニアになれる」は本当だが、**どんな企業にどんな条件で就職できるか**はスキルと努力次第だ。

## 未経験者向けロードマップ

### Step 1: 目標設定（1〜2週間）

転職活動を始める前に、以下を明確にしよう。

**目指すエンジニア職種**:
- Webエンジニア（フロントエンド/バックエンド/フルスタック）
- インフラエンジニア（クラウド/ネットワーク/SRE）
- モバイルエンジニア（iOS/Android）
- データエンジニア/AIエンジニア

**就職したい企業タイプ**:
- 自社開発（Web系スタートアップ〜大企業）
- 受託開発（クライアントの依頼を受けて開発）
- SES（客先常駐型のシステムエンジニアリングサービス）

目標を定めることで、学ぶべき技術が絞られ、効率よく準備できる。

---

### Step 2: 基礎スキル習得（3〜6ヶ月）

#### Webエンジニア志望の学習ロードマップ

```
Month 1-2: Web基礎
├── HTML/CSS（マークアップ・レイアウト）
├── JavaScript（基礎文法・DOM操作）
└── Git/GitHub（バージョン管理）

Month 3-4: バックエンド
├── Python または Ruby
├── Djangoまたは Ruby on Rails
└── データベース（SQL/PostgreSQL）

Month 5-6: 実践・ポートフォリオ
├── オリジナルアプリ開発（3〜5本）
├── AWS/GCP入門
└── Docker入門
```

#### インフラエンジニア志望の学習ロードマップ

```
Month 1-2: ネットワーク・OS基礎
├── Linux コマンド操作
├── ネットワーク基礎（TCP/IP、HTTP）
└── 仮想化技術（VirtualBox/VMware）

Month 3-4: クラウド入門
├── AWS基礎（EC2/S3/RDS/VPC）
├── Docker・コンテナ技術
└── Terraform入門（IaC）

Month 5-6: 資格・実践
├── AWS SAA（ソリューションアーキテクト アソシエイト）
├── 個人でAWS環境構築
└── 簡単なCI/CD構築
```

### Step 3: ポートフォリオ作成（1〜2ヶ月）

未経験者にとってポートフォリオは**最大の武器**だ。採用担当者はポートフォリオを見て「この人はエンジニアとして働けるか」を判断する。

#### 良いポートフォリオの条件

1. **動くものを作る**: 机上の勉強ではなく、実際に動くアプリを作ること
2. **GitHubに公開**: コードの品質・コミット頻度を見られる
3. **README を丁寧に書く**: 技術スタック・機能・使い方を明記
4. **オリジナリティがある**: Todoアプリだけでは他の応募者と差別化できない

#### おすすめポートフォリオアイデア

| 難易度 | アイデア | 使用技術例 |
|---|---|---|
| ★☆☆ | 個人ブログ・ポートフォリオサイト | HTML/CSS/JavaScript |
| ★★☆ | ToDoアプリ（CRUD機能） | React + Node.js |
| ★★☆ | 家計簿アプリ | Vue.js + Django |
| ★★★ | ECサイト | Rails + PostgreSQL + Stripe |
| ★★★ | リアルタイムチャット | WebSocket + React + Node.js |

## 経験者向けロードマップ

### 現役エンジニアが転職するタイミング

エンジニアの転職適齢期は、実は「いつでも」だ。ただし、以下のタイミングは特に動きやすい。

- **3年目〜5年目**: 基礎スキルが固まり、次のキャリアを考え始める時期
- **技術スタックの変化期**: 担当技術が変わったとき、得意技術で別の企業を探す
- **プロジェクト区切り**: 大型プロジェクト完了後、次のチャレンジを求める

## 技術面接対策チェックリスト

### コーディングテスト

- [ ] LeetCode Easy問題を100問以上解いている
- [ ] Medium問題も30問以上解いている
- [ ] 時間制限内（30〜45分）でコードが書ける

### システム設計面接

- [ ] スケーラブルなシステム設計の基礎を理解している
- [ ] CAP定理、データベース選択基準を説明できる
- [ ] 実際の大規模サービス（Twitter、YouTube等）の設計を考えてみた

### 行動面接

- [ ] STARメソッド（状況・課題・行動・結果）で過去の経験を話せる
- [ ] 「失敗した経験とそこから学んだこと」を準備している
- [ ] 志望動機が具体的で説得力がある

## 職務経歴書の書き方【NG例・OK例で解説】

エンジニア転職において、職務経歴書は書類選考を通過するための最重要書類だ。特に未経験者は書き方ひとつで結果が大きく変わる。

### NG例とOK例

| | NG例 | OK例 |
|---|---|---|
| 技術 | 「ReactとRailsを学びました」 | 「React/TypeScript + Railsを6ヶ月学習し、タスク管理アプリを開発・デプロイ（GitHub: コミット数280）」 |
| 強み | 「コミュニケーション能力に自信があります」 | 「前職（営業）の要件ヒアリング力を活かし、チーム開発でユーザーストーリー作成を担当」 |
| 意欲 | 「学ぶ意欲は誰にも負けません」 | 「5人チームで2週間スプリントを4回実施し、計画通りにリリースを完了」 |

### 経験者の職務経歴書で差がつくポイント

| 項目 | 弱い書き方 | 強い書き方 |
|---|---|---|
| 技術 | 「Java、Python使用経験あり」 | 「Java（5年）でマイクロサービス設計、Python（3年）でデータパイプライン構築」 |
| 実績 | 「大規模システム開発に参画」 | 「月間500万PVのECサイトAPI設計を担当、レスポンスタイムを40%改善」 |
| 役割 | 「チームリーダーとして活躍」 | 「5名チームのテックリードとしてコードレビュー・技術選定を担当」 |

数字で語ることが最重要だ。「大規模」「多数」ではなく、具体的な数値を記載しよう。

## 企業タイプ別 技術面接で聞かれる質問

面接で聞かれる質問は企業タイプによって大きく異なる。事前に対策しておくことで、通過率を大幅に向上させられる。

### Web系自社開発企業（メガベンチャー・スタートアップ）

- 「このAPIのレスポンスタイムが遅い原因を3つ挙げて、それぞれの解決策を説明してください」
- 「100万ユーザーが同時アクセスするサービスのアーキテクチャを設計してください」
- 「直近のプロジェクトで最も難しかった技術的課題と、どう解決したかを教えてください」

### 受託開発企業（SIer含む）

- 「クライアントの要件が曖昧なとき、どのように整理しますか？」
- 「納期に間に合わないと判断した場合、どう対処しますか？」
- 「ウォーターフォールとアジャイル、それぞれのメリット・デメリットを説明してください」

### SES企業

- 「直近1ヶ月で学んだ技術は何ですか？」
- 「どのような開発環境で作業した経験がありますか？」
- 「チームで開発する際に大切にしていることは何ですか？」

## 年収交渉の戦略

### 未経験者の年収交渉

未経験者は「年収より成長環境」を優先すべきだが、交渉の余地がないわけではない。

| 企業タイプ | 年収レンジ | 交渉余地 |
|---|---|---|
| SES企業 | 280万〜350万円 | 小 |
| 受託開発 | 300万〜400万円 | 中 |
| 自社開発（中小） | 350万〜450万円 | 中 |
| 自社開発（大手） | 400万〜500万円 | 大 |

**交渉材料**: 前職での実績、質の高いポートフォリオ、関連資格（AWS SAA等）、他社からの内定（最も強力）。

### 経験者の年収交渉テクニック

1. **現年収ではなく市場価値で交渉する**: 転職サイトの年収診断や他社オファーを根拠にする
2. **オファー面談を活用する**: 内定後のオファー面談が年収交渉の最適タイミング
3. **複数社を並行で進める**: 他社オファーがあると交渉力が上がる
4. **年収以外も交渉する**: リモートワーク、副業可否、書籍購入補助なども重要な待遇

## 転職でよくある失敗パターン5選

### 失敗1: ポートフォリオが「スクール課題の延長」

スクールのカリキュラム通りに作ったアプリだけでは差別化できない。オリジナル機能の追加や、完全に独自のアプリを作ることが重要だ。

### 失敗2: 1社しか応募しない

1社に絞ると不採用時のダメージが大きく交渉力も持てない。最低10社以上に応募しよう。

### 失敗3: 志望動機が弱い

面接官が最も知りたいのは「なぜうちの会社なのか」だ。企業のプロダクト・技術ブログ・カルチャーを調べた上で、具体的な志望理由を準備しよう。

### 失敗4: 退職してから転職活動を始める

収入が途切れるリスクを避けるためにも在職中に始めるべきだ。空白期間が長いと面接での印象もマイナスになる。

### 失敗5: 年収だけで転職先を決める

「技術的に成長できる環境か」「チームのカルチャーが合うか」も長期的なキャリアには不可欠だ。特に未経験者は、最初の1〜2年でどれだけ成長できるかが今後のキャリアを左右する。

## まとめ

エンジニア転職は「準備8割・運2割」だ。

**未経験者へ**: スキル習得に6ヶ月〜1年、ポートフォリオ作成に1〜2ヶ月の準備期間を確保しよう。焦らず着実にスキルを積み上げることが成功への近道だ。

**経験者へ**: 現在の市場価値を客観的に把握し、次のキャリアステップを明確にしてから転職活動を開始しよう。転職エージェントを活用して、非公開求人も含めた幅広い選択肢を検討することをおすすめする。
