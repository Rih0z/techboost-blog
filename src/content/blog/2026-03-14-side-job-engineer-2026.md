## はじめに

「副業を始めたいけど、エンジニアスキルをどう活かせばいいかわからない」という声をよく聞く。

2026年現在、副業解禁が進む中でITエンジニアの副業市場は急拡大している。スキルさえあれば月3万〜20万円の副収入は十分に現実的だ。

この記事では、副業エンジニアの実態から案件の取り方、税金の注意点まで、ロードマップ形式で解説する。

## 2026年 エンジニア副業市場の現状

### 副業を認める企業の増加

厚生労働省の調査によると、2025年時点で副業・兼業を認める企業は大企業で約65%、中小企業でも40%超に達している。2026年はさらに普及が進む見通し。

### 副業エンジニアの平均収入

| 経験年数 | 週稼働時間 | 月収目安 |
|---------|-----------|--------|
| 1〜2年 | 5〜10時間 | 3万〜8万円 |
| 3〜5年 | 10〜20時間 | 8万〜15万円 |
| 5年以上 | 20〜30時間 | 15万〜30万円 |
| 10年以上（上流工程） | 15〜20時間 | 30万〜60万円 |

## 副業案件の4種類

### 1. 受託開発（最もポピュラー）

クライアントから依頼を受けてWebアプリ・モバイルアプリ・ツールを開発する形態。

**メリット**
- スキルに直結した高単価案件が多い
- ポートフォリオとして成果物を活用できる
- 自分のペースで作業できる

**デメリット**
- 納期管理・顧客対応が必要
- スコープ外の作業を求められるリスク
- 最初の案件獲得が難しい

**向いているスキル**: Web開発（React/Vue/Next.js）、バックエンド（Node.js/Python/Go）、スマートフォンアプリ（Swift/Kotlin/Flutter）

### 2. SES的な副業（週10〜20時間の定期稼働）

特定企業のプロジェクトに週数回参加する形態。準委任契約が多い。

**メリット**
- 安定した収入が見込める
- 長期契約によるリレーション構築
- チームで働く環境を副業でも体験できる

**デメリット**
- 副業禁止の本業企業との競合リスク
- 週のスケジュール調整が必要
- 突発的な変更への対応が求められることも

**月額単価目安**

| スキル | 週稼働 | 月額相場 |
|-------|-------|--------|
| フロントエンド（React/Vue） | 15時間 | 10万〜18万円 |
| バックエンド（Python/Go） | 15時間 | 12万〜22万円 |
| インフラ（AWS/GCP） | 10時間 | 12万〜25万円 |
| スクラムマスター | 10時間 | 15万〜30万円 |

### 3. 教育・メンタリング

プログラミングスクールの講師や、個人に対してのメンタリングを行う形態。

**メリット**
- 自分の知識の整理・体系化につながる
- 固定スケジュールで安定収入
- 教える経験がエンジニアとしての成長にも貢献

**デメリット**
- 単価が受託・SES案件より低いことが多い
- 受講生のレベルに合わせた対応が必要

**収入例**
- プログラミングスクール講師: 時給2,500〜5,000円
- 個人メンタリング: 1回60分 5,000〜15,000円
- オンライン教材販売: 月0〜50万円（不労所得型）

### 4. OSSコントリビューション・スポンサーシップ

オープンソースプロジェクトに貢献することで、企業からスポンサー支援を受ける形態。

**メリット**
- 実力があれば高収入も可能（月$1,000〜10,000以上）
- 国際的なコミュニティへの貢献
- GitHub上での実績がポートフォリオになる

**デメリット**
- 収入が安定しない
- 一定の知名度と実績が必要
- 長期的な取り組みが必要

### SNS・直接営業での案件獲得

2026年現在、SNSを活用した直接営業が増加している。

**Twitter/X（最も効果的）**
- 技術記事の投稿で専門性をアピール
- エンジニアコミュニティへの参加で人脈形成
- 「副業相談DM歓迎」とプロフィールに記載するだけで問い合わせが来ることも

**LinkedIn**
- 外資系・グローバル案件の獲得に有効
- プロフィールを英語で書くと海外案件の窓口になる
- 自動メッセージでのスパム的なアプローチには注意

**技術ブログ（長期戦略）**
- Zennやnote、個人ブログで技術情報を発信
- 検索流入で案件依頼が自然に来る状態を作る
- 短期間での収益化は難しいが、長期的な資産になる

## 副業エンジニアの実践ロードマップ

### Phase 1: 副業準備（1〜2ヶ月目）

```
[ ] ポートフォリオサイトを作成（GitHub Pages or Vercel）
[ ] GitHubに公開リポジトリを5〜10個用意
[ ] 技術ブログを始める（週1本ペース）
[ ] クラウドワークス・ランサーズにプロフィール登録
[ ] 副業用の銀行口座・クレジットカードを開設
```

**ポートフォリオに含めるべき内容**
- 自己紹介と得意技術スタック
- 実績プロジェクト（本業・個人開発を問わず）
- 技術ブログへのリンク
- 連絡先（メール + SNS）

### Phase 2: 最初の案件獲得（2〜4ヶ月目）

```
[ ] クラウドソーシングで低単価案件を受注（実績作り）
[ ] SNSで技術情報の発信を継続
[ ] 技術コミュニティに参加・貢献
[ ] 最初の5〜10件の実績を積む
[ ] 実績をポートフォリオに追記
```

**最初の案件で注意すること**
- 単価より「実績作り」を優先
- 小規模なタスク（ランディングページ制作・API作成など）から始める
- 契約書・発注書の取り交わしを必ず行う

### Phase 3: 単価アップ・安定化（4〜12ヶ月目）

```
[ ] 実績・レビューを積み上げ単価交渉
[ ] 得意分野を1〜2つに絞って専門家としてポジショニング
[ ] 直接営業・SNS経由の案件獲得にシフト
[ ] 月10万円以上を安定して稼ぐ基盤を構築
[ ] 長期継続案件（3ヶ月以上）の確保
```

## 税金・確定申告の注意点

副業収入が年間20万円を超えると、確定申告が必要になる。

### 副業収入の税金計算

| 年収（本業＋副業） | 税率（所得税） |
|-----------------|------------|
| 〜195万円 | 5% |
| 195〜330万円 | 10% |
| 330〜695万円 | 20% |
| 695〜900万円 | 23% |
| 900〜1,800万円 | 33% |
| 1,800〜4,000万円 | 40% |
| 4,000万円超 | 45% |

### 住民税の申告方法

副業収入がある場合、住民税の「普通徴収」を選択することが重要。

- **特別徴収のまま**: 副業収入分の住民税が本業の給与天引きに上乗せされ、副業がバレる可能性がある
- **普通徴収**: 副業分の住民税は自分で納付するため、本業の会社に通知が行かない

```
確定申告書の「住民税・事業税に関する事項」欄で
「自分で納付」を選択する
```

### 経費として計上できるもの

副業に使った費用は経費として計上できる。

| 経費項目 | 説明 | 計上率の目安 |
|--------|-----|-----------|
| PC・周辺機器 | 副業に使用する割合で按分 | 50〜100% |
| 通信費（インターネット） | 副業使用割合で按分 | 20〜50% |
| 書籍・学習費 | 副業に直結するもの | 100% |
| ソフトウェア費用 | 副業で使うSaaS等 | 100% |
| 自宅の一部（家賃） | 副業作業スペース分 | 10〜30% |

### 個人事業主として開業するメリット

年間副業収入が100万円を超えてきたら、個人事業主として開業届を出すことを検討しよう。

**開業するメリット**
- 青色申告で最大65万円の特別控除が受けられる
- 家族への給与（青色事業専従者給与）が経費になる
- 赤字を3年間繰り越せる

**開業に必要な手続き**
1. 税務署に「個人事業の開業届出書」を提出
2. 青色申告承認申請書を同時に提出（青色申告希望の場合）
3. 開業freeeや弥生などの会計ソフト導入を検討

## 副業エンジニアの収入ステップアップ戦略

### 単価アップの3つの方法

**1. 専門分野の深掘り**

「何でもできる」より「〇〇の専門家」として認知されると単価が上がる。

高単価になりやすい専門分野:
- AI/機械学習エンジニアリング
- セキュリティ（ペネトレーションテスト・脆弱性診断）
- クラウドインフラ設計（AWS/GCP Solutions Architect相当）
- DX推進コンサルティング

**2. 上流工程へのシフト**

要件定義・システム設計・技術選定など、上流工程の作業は実装よりも高単価になる。

**3. 時間単価から成果報酬へ**

固定時間単価から、プロジェクト完了報酬や成果連動報酬に移行することで、効率が上がるほど実質時間単価が上がる。

## よくある疑問

**Q: 本業の会社に副業がバレますか？**

A: 住民税の申告を「普通徴収」にすれば、会社への通知は原則ありません。ただし副業禁止規定がある会社では、万が一バレた場合のリスクを考慮しておく必要があります。

**Q: 副業始めに向いているスキルは？**

A: WordPressサイト制作、LP制作、ReactやVueでのWebアプリ開発が需要が高く始めやすいです。AIやデータ分析スキルは単価が高い傾向にあります。

**Q: 週5時間未満でも副業できますか？**

A: 可能ですが、受託開発より教材販売やOSS活動など、非同期で進めやすい形態が向いています。
