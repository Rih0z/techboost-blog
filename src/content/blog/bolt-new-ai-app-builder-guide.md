# Bolt.new完全ガイド｜プロンプトだけでフルスタックアプリを構築するAIツール

社内ツールのプロトタイプをBolt.newで作ったら、企画会議の翌日にはデモ可能な状態でした。「こんなツールがほしい」という口頭のアイデアを、その場でプロンプトに落とし込み、ブラウザ上で動くアプリケーションとしてチームに見せる。従来なら環境構築だけで半日かかっていた作業が、ものの数分で完了した体験は衝撃的でした。

この記事では、StackBlitzが開発したAIアプリビルダー「Bolt.new」について、技術的な仕組みから実践的な使い方、料金プラン、競合ツールとの比較まで徹底解説します。

---

## Bolt.newとは？

Bolt.newは、StackBlitzが提供するAI搭載のフルスタックアプリケーションビルダーです。ブラウザ上でプロンプト（自然言語の指示）を入力するだけで、フロントエンドからバックエンドまで含むWebアプリケーションを自動生成し、その場でプレビュー・編集・デプロイまで完結できます。

### 従来のアプリ開発との違い

従来のフルスタック開発では、以下のような手順が必要でした。

```
従来のワークフロー:
1. Node.js / npm のインストール
2. フレームワークの選定とプロジェクト初期化
3. ディレクトリ構成の設計
4. 依存パッケージのインストール
5. ルーティング・状態管理・API設計
6. コーディング
7. ローカルでのデバッグ
8. デプロイ設定・CI/CD構築
```

Bolt.newでは、これらのステップがすべてブラウザ内で自動化されます。

```
Bolt.newのワークフロー:
1. プロンプトを入力（「タスク管理アプリを作って」）
2. AIがコード生成 + 依存関係解決 + ビルド
3. ブラウザ内でリアルタイムプレビュー
4. 必要に応じて追加プロンプトで修正
5. ワンクリックでデプロイ
```

ローカル環境を一切汚さずに、アイデアからデプロイまでをブラウザだけで完結できる点が最大の特徴です。

### 主な特徴

| 特徴 | 説明 |
|------|------|
| **ブラウザ完結** | インストール不要。ブラウザだけでフルスタック開発が可能 |
| **AIコード生成** | 自然言語プロンプトからReact/Next.js/Astro等のコードを自動生成 |
| **リアルタイムプレビュー** | 生成されたアプリをその場で動作確認 |
| **パッケージ管理** | npm依存関係を自動解決。手動でのinstall不要 |
| **ワンクリックデプロイ** | Netlifyへの即時デプロイに対応 |
| **コード編集** | 生成コードをエディタ上で自由に編集可能 |
| **会話型UI** | チャットで追加の修正・機能追加を指示 |

## WebContainers技術の解説

Bolt.newの根幹を支えるのが、StackBlitzが独自開発した**WebContainers**テクノロジーです。

### WebContainersとは

WebContainersは、ブラウザ内でNode.jsランタイムを完全に実行する技術です。通常、Node.jsはサーバーサイドのOS上で動作しますが、WebContainersはWebAssembly（Wasm）を活用して、ブラウザのサンドボックス内にNode.js互換の実行環境を構築します。

```
従来のクラウドIDE:
[ブラウザ] ←→ [リモートサーバー上のNode.js]
  遅延あり     サーバーコスト発生

WebContainers:
[ブラウザ内でNode.jsが直接動作]
  遅延なし     サーバーコスト不要
```

### 技術的なアーキテクチャ

WebContainersが実現していることを技術的に分解すると、以下の要素から構成されています。

```
WebContainers アーキテクチャ:
├── WebAssemblyベースのNode.jsランタイム
│   ├── npm / pnpm パッケージマネージャ
│   ├── ファイルシステム（仮想）
│   └── プロセス管理
├── ネットワークスタック
│   ├── Service Worker によるHTTPリクエスト処理
│   └── ローカルサーバーのエミュレーション
├── ターミナルエミュレータ
│   └── シェルコマンド実行
└── ブラウザサンドボックス
    └── セキュリティ境界
```

### なぜWebContainersが重要なのか

1. **レイテンシーの排除**: リモートサーバーとの通信が不要なため、ファイル保存からプレビュー反映までミリ秒単位で完了します
2. **セキュリティ**: ブラウザのサンドボックス内で動作するため、ホストOSへのアクセスが物理的に不可能です
3. **コスト効率**: サーバーインスタンスを起動する必要がないため、同時利用者が増えてもインフラコストが線形に増加しません
4. **オフライン対応**: 初回ロード後はネットワーク接続なしでも動作が可能です

従来のクラウドIDEが抱えていた「起動に30秒〜数分かかる」「サーバーがスリープする」といった問題を根本的に解決した技術と言えます。

## 使い方：プロンプトからデプロイまで

### Step 1: アクセスとサインアップ

ブラウザで [bolt.new](https://bolt.new) にアクセスします。Googleアカウント、GitHubアカウント、またはメールアドレスでサインアップが可能です。無料プランでも基本機能は利用できます。

### Step 2: プロンプトを入力

画面中央のプロンプト入力欄に、作りたいアプリケーションの説明を自然言語で入力します。

```
プロンプト例:
「Reactで社内タスク管理ツールを作ってください。
 - タスクの追加・編集・削除ができる
 - ステータス（未着手/進行中/完了）で管理
 - ドラッグ&ドロップでステータスを変更できるカンバンボード
 - ダークモード対応
 - Tailwind CSSでスタイリング」
```

### Step 3: コード生成とプレビュー

プロンプト送信後、AIがリアルタイムでコードを生成していきます。画面は左右に分割され、左側にファイルツリーとコードエディタ、右側にアプリケーションのライブプレビューが表示されます。

生成中もプレビューはリアルタイムで更新されるため、AIがどのようにアプリを組み立てているかを視覚的に確認できます。

### Step 4: 追加指示による修正

生成されたアプリに対して、チャット形式で追加の指示を出すことができます。

```
修正プロンプト例:
「タスクカードにラベル（優先度：高/中/低）を追加して、
ラベルごとに色分けしてください。
高は赤、中は黄色、低は青で表示します。」
```

AIは既存のコードコンテキストを理解しているため、ゼロから再生成するのではなく、該当箇所のみを修正します。

### Step 5: デプロイ

アプリが完成したら、画面右上の「Deploy」ボタンをクリックします。Netlifyへのデプロイがワンクリックで完了し、公開URLが発行されます。

```
デプロイの流れ:
1. 「Deploy」ボタンをクリック
2. Netlifyアカウントと連携（初回のみ）
3. ビルド → デプロイ → URLが発行される
4. 発行されたURLをチームに共有
```

## Supabase / Firebase連携

Bolt.newは、バックエンドサービスとの連携もプロンプトで指示できます。

### Supabase連携

Supabaseは、PostgreSQLベースのオープンソースBaaS（Backend as a Service）です。Bolt.newのプロンプトでSupabaseを指定すると、認証・データベース・リアルタイムサブスクリプションの接続コードが自動生成されます。

```
連携プロンプト例:
「Supabaseをバックエンドに使ったTodoアプリをReactで作成してください。
 - Supabaseの認証（メール/パスワード + Google OAuth）
 - todosテーブルへのCRUD操作
 - リアルタイム更新（Supabase Realtime）
 - Row Level Security（ユーザーごとのデータ分離）

SupabaseのURLとAnon Keyは環境変数(.env)から読み込む設計にしてください。」
```

生成されるコードには `@supabase/supabase-js` のセットアップ、型定義、RLSを考慮したクエリが含まれます。ただし、Supabase側のテーブル作成やRLSポリシー設定はSupabaseダッシュボード上で手動対応が必要です。

### Firebase連携

Firebase連携も同様にプロンプトで指示可能です。

```
連携プロンプト例:
「Firebaseを使ったリアルタイムチャットアプリをNext.jsで作成してください。
 - Firebase Authentication（Google ログイン）
 - Cloud Firestoreでメッセージ保存
 - onSnapshotによるリアルタイム更新
 - Firebase Hostingへのデプロイ設定

FirebaseConfigは環境変数から読み込む設計にしてください。」
```

### 連携時の注意点

- **APIキーの取り扱い**: 生成コードにAPIキーがハードコードされる場合があるため、必ず環境変数に移動してください
- **セキュリティルール**: Supabase RLS / Firebase Security Rules は自動生成されないことが多いため、手動で設定が必要です
- **無料枠の確認**: Supabase / Firebase ともに無料枠に制限があるため、本番利用前に確認しましょう

## デプロイ方法

Bolt.newで生成したアプリをデプロイする方法は複数あります。

### 1. Netlify（ワンクリック）

Bolt.newの画面から直接Netlifyにデプロイできます。最も手軽な方法です。

```
手順:
1. 画面右上の「Deploy」ボタンをクリック
2. Netlifyアカウントでログイン（初回のみ）
3. 自動でビルド＆デプロイ

```

2. Vercelダッシュボードで該当リポジトリをインポート

## Bolt.new vs v0.dev vs Replit Agent vs Lovable 比較

Bolt.newと同じ領域で競合するAIコード生成ツールを比較します。

| 項目 | **Bolt.new** | **v0.dev** | **Replit Agent** | **Lovable** |
|------|-------------|-----------|-----------------|------------|
| **提供元** | StackBlitz | Vercel | Replit | Lovable (旧GPT Engineer) |
| **得意領域** | フルスタックアプリ全般 | UIコンポーネント・フロントエンド | フルスタック + データベース | フルスタックアプリ |
| **実行環境** | WebContainers（ブラウザ内） | サーバーサイド生成 | クラウドVM | クラウドサーバー |
| **プレビュー** | リアルタイム（ブラウザ内） | 静的プレビュー | リアルタイム（クラウド） | リアルタイム（クラウド） |
| **バックエンド** | Node.js（ブラウザ内） | フロントのみ（基本） | Python / Node.js | Node.js |
| **デプロイ** | Netlify | Vercel | Replit Hosting | Lovable Hosting |
| **DB連携** | Supabase / Firebase | なし（手動） | PostgreSQL内蔵 | Supabase |
| **コード所有** | 全コードアクセス可 | コードコピー可 | 全コードアクセス可 | 全コードアクセス可 |
| **無料枠** | あり（日次制限） | あり（月次制限） | あり | あり（月次制限） |
| **月額（Pro）** | $20 | $20 | $25 | $20 |
| **起動速度** | 数秒（ブラウザ内） | 数秒 | 30秒〜1分（VM起動） | 数秒〜10秒 |
| **最適な用途** | フルスタックプロトタイプ | UIデザイン→コード変換 | 本格的なアプリ開発 | 非エンジニア向けアプリ構築 |

### 使い分けの指針

- **UIコンポーネントだけ欲しい** → v0.dev が最適。shadcn/ui準拠の高品質なコンポーネントを生成します
- **フルスタックプロトタイプを最速で** → Bolt.new が最適。ブラウザ内完結で起動が速い
- **本格開発をクラウドで継続** → Replit Agent が最適。DBやシークレット管理が内蔵されている
- **非エンジニアが自分でアプリを作りたい** → Lovable が最適。よりガイド付きのUXを提供

## 制限事項と注意点

Bolt.newは強力なツールですが、万能ではありません。以下の制限事項を理解した上で活用してください。

### 複雑なアプリケーションの限界

- **大規模なファイル数**: 数十ファイルを超えるプロジェクトでは、AIの文脈理解が追いつかなくなることがあります。修正指示が既存のコードと矛盾するケースが発生しがちです
- **複雑なビジネスロジック**: 複雑な計算処理、マルチステップのワークフロー、条件分岐の多いバリデーションなどは、プロンプトだけでは正確に伝えきれない場合があります
- **パフォーマンス最適化**: 生成コードはまず「動くこと」を優先するため、メモ化、遅延ロード、バンドルサイズ最適化などは手動でチューニングが必要です

### WebContainers固有の制限

- **ネイティブモジュール**: C/C++拡張を含むnpmパッケージ（例: `bcrypt`, `sharp`など）はWebContainers上では動作しません。JavaScript/Wasm実装の代替パッケージ（`bcryptjs`, `sharp-wasm`等）を使う必要があります
- **ファイルシステム**: 仮想ファイルシステムであるため、大容量ファイルの読み書きやディスクI/Oに制約があります
- **長時間プロセス**: WebSocketサーバーやバックグラウンドジョブなどの長時間実行プロセスはブラウザのライフサイクルに依存します

### 本番利用に向けた注意

- **生成コードのレビュー**: AIが生成したコードをそのまま本番に投入するのは推奨されません。セキュリティ、エラーハンドリング、エッジケースの考慮が不十分な場合があります
- **テストの追加**: 生成コードにはテストが含まれないことが多いため、本番利用前にテストコードを追加してください
- **依存パッケージの監査**: 自動インストールされたパッケージのバージョンやライセンスを確認しましょう
- **認証情報の管理**: 環境変数に移動すべきAPIキーやシークレットがコードにハードコードされていないか確認してください

## どんな時にBolt.newを使うべきか — 判断基準

Bolt.newが特に威力を発揮するシーンと、別のアプローチが適しているシーンを整理します。

### Bolt.newが最適なケース

| シーン | 理由 |
|--------|------|
| **プロトタイプの高速作成** | 企画会議後にすぐデモを作りたい場合、ブラウザだけで数分〜数十分で動くものが見せられる |
| **社内ツールのPoC** | 本格開発の前に「こういうものが欲しい」を可視化する用途に最適 |
| **ハッカソン・アイデアソン** | 時間制限がある環境で、素早くアイデアを形にする必要がある |
| **フロントエンドの学習** | 「こう書けばこう動く」をインタラクティブに体験できる教材として |
| **クライアントへのデモ** | 要件定義段階で動くモックを見せ、認識のすり合わせに活用 |
| **技術選定の比較** | 同じアプリをReact版とVue版で作り、比較検討する |

### 別のアプローチが適しているケース

| シーン | 推奨アプローチ |
|--------|--------------|
| **大規模な本番アプリケーション** | ローカル開発環境 + CI/CD。Bolt.newで作ったプロトタイプをベースにローカルへ移行 |
| **既存コードベースへの機能追加** | GitHub Copilot / Cursor。既存のコンテキストを理解した上でのコード補完が得意 |
| **チーム開発（5人以上）** | GitHubワークフロー + PR レビュー。Bolt.newのコラボ機能は小規模チーム向け |
| **ネイティブモジュール依存の処理** | ローカル環境。画像処理（sharp）、暗号化（bcrypt）などWebContainers非対応の機能 |
| **マイクロサービスアーキテクチャ** | Docker Compose / Kubernetes。複数サービスの連携はBolt.newの範囲外 |

### 実務での使い分けフロー

```
アイデア・要件がある
  │
  ├── 「動くものを見たい」→ Bolt.new でプロトタイプ
  │     │
  │     ├── クライアントOK → コードをエクスポート → ローカル開発へ移行
  │     └── 方向性変更 → Bolt.new で別パターン生成
  │
  ├── 「UIだけ欲しい」→ v0.dev でコンポーネント生成
  │
  └── 「本番コードを書きたい」→ ローカル環境 + GitHub Copilot / Cursor
```

重要なのは、**Bolt.newを「最終成果物を作るツール」ではなく「アイデアを素早く検証するツール」として位置づける**ことです。生成されたコードをそのまま本番にデプロイするのではなく、プロトタイプとして活用し、本番開発ではコードレビュー・テスト追加・セキュリティ監査を経て品質を担保するワークフローが推奨されます。

## まとめ

Bolt.newは、StackBlitzのWebContainers技術をベースに、プロンプト入力だけでフルスタックアプリケーションを生成・プレビュー・デプロイできるAIツールです。

### この記事のポイント

- **ブラウザ完結**: WebContainersにより、ローカル環境構築なしでNode.jsアプリがブラウザ内で動作する
- **高速プロトタイピング**: プロンプトからデプロイまで数分。企画段階のアイデアを即座に可視化できる
- **フレームワーク対応**: React、Next.js、Remix、Astro、Vue、Svelteなど主要フレームワークに対応
- **バックエンド連携**: Supabase / Firebaseとの連携コードも自動生成可能
- **競合との差別化**: WebContainersによるブラウザ内実行が最大の強み。起動速度とレイテンシーで優位
- **適材適所**: プロトタイプ・PoC・デモに最適。本番アプリケーションはエクスポート後にローカル開発へ移行

フルスタック開発のハードルを劇的に下げるBolt.newは、エンジニアの「アイデアから動くもの」までの距離を縮める強力なツールです。まずは無料プランで1つアプリを作ってみると、その速度感に驚くはずです。
